# UI Principles

Health Pilot is not designed to maximize engagement.

It is designed to minimize cognitive effort.

Every screen exists to help people make one better health decision.

---

# Core Principle

Mission first.

Everything else supports the mission.

The user should know what to do within a few seconds of opening the app.

---

# One Screen, One Decision

Each screen should answer one primary question.

Examples:

- What should I do now?
- Why is this today's mission?
- How am I improving?

Avoid presenting multiple competing actions.

---

# Show Actions Before Data

Never lead with metrics.

Instead of:

HRV: 54 ms

Sleep Score: 81

Recovery: 72

Show:

Today's Mission

Walk outside for 20 minutes before 10:00.

Metrics exist to support the recommendation.

Not the other way around.

---

# Progressive Disclosure

Show only what is needed.

Reveal additional information only when users ask for it.

Recommended order:

Mission

↓

Reason

↓

Health metrics

↓

Raw data

---

# One Primary Action

Every screen should have one obvious primary action.

Examples:

Complete Mission

View Reason

Start Walk

Avoid multiple competing buttons.

---

# Reduce Visual Noise

Remove anything that does not help decision making.

Avoid:

- unnecessary cards
- decorative charts
- duplicated information
- excessive colors
- persistent notifications

Whitespace is part of the interface.

---

# Use Numbers Carefully

Numbers are evidence.

They are not the product.

Never expect users to understand health metrics before taking action.

Translate numbers into recommendations.

---

# AI Explains, People Decide

Health Pilot recommends.

Users decide.

The interface should never pressure users into following recommendations.

Explain reasoning clearly.

Respect user autonomy.

---

# Design for Seconds

Most interactions should require less than five seconds.

Users should understand:

- today's mission
- why it matters
- what to do next

without reading long explanations.

---

# Calm by Default

Health is a long-term journey.

The interface should feel calm, confident, and supportive.

Avoid urgency unless immediate action is truly required.

Design should reduce anxiety, not create it.

---

# Every Pixel Must Earn Its Place

Every component should answer one question:

"Does this help the user make a better health decision?"

If not,

remove it.
